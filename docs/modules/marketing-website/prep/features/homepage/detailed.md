## 2. Business Rules

1. **Homepage Content Ownership**
   - Homepage content is managed by product/marketing team
   - Copy can be updated without deploying code (optional: CMS integration)
   - Value proposition messaging does not change per user segment (same page for all)
   - Persona-specific messaging is sections on same page, not different pages

2. **Call-to-Action Behavior**
   - "Start Free Trial" button links to signup form (on same page via anchor link or modal, or to /signup page)
   - "For Institutions" link directs to /institutions or /colleges page (separate page or section)
   - All external links (Privacy, Terms) open in new tab/window
   - Internal links (signup, institutions) navigate within site

3. **Mobile Responsiveness**
   - All content must be readable and functional on screens as small as 375px (mobile) to 1920px (desktop)
   - Typography scales appropriately: headlines smaller on mobile
   - Images and icons scale proportionally
   - Buttons are tappable on mobile (min 44x44px touch target)
   - Spacing/padding reduces on mobile to avoid excessive scrolling

4. **SEO Optimization**
   - Page title: "NicheLearner - Personalized JEE Prep Assessment & Guidance"
   - Meta description: "Adaptive quizzes, honest feedback, and cross-topic guidance for JEE Mains. Free 30-day trial, no credit card required."
   - Heading hierarchy: One H1 (main headline), multiple H2s (section titles), H3s for subsections
   - Alt text on all images and icons
   - No keyword stuffing; natural, readable copy

5. **Analytics & Tracking**
   - Google Analytics events tracked:
     - "homepage_view" — when page loads
     - "homepage_cta_click" — when "Start Free Trial" button clicked
     - "homepage_college_link_click" — when "For Institutions" link clicked
     - "homepage_signup_modal_open" — if signup form embedded as modal
   - UTM parameters preserved and passed to signup form
   - No personally identifiable information in analytics

6. **Internationalization** (Future consideration)
   - Page currently English-only
   - Structure allows for future language translations (Hindi, etc.)
   - No hardcoded text in HTML; all copy defined in configuration or CMS

7. **Accessibility (WCAG 2.1 AA Compliance)**
   - All text has sufficient contrast (4.5:1 minimum for body text)
   - Headings are semantic (H1, H2, H3)
   - Images have alt text
   - Links are distinguishable (not color-only, underlined or icon)
   - No auto-playing videos or animations that distract
   - Form inputs (if signup on homepage) have labels

## Edge Cases

1. **User Disables JavaScript**
   - Signup button shows "JavaScript required to sign up"
   - Static content (value prop, trust markers) still visible
   - Links to /signup page work without JS

2. **User on Very Slow Network**
   - Page loads static HTML first, images lazy-load
   - Hero section text visible before images load
   - Signup button functional immediately, form loads progressively

3. **User Views Homepage on 4:3 Aspect Ratio (Tablet)**
   - Layout adapts: 2-column cards instead of 3
   - Text size and padding adjust
   - Buttons remain tappable size

4. **User Resizes Window While on Homepage**
   - Page responsive and reflows smoothly
   - No content breaks or overlaps at any breakpoint

5. **Homepage Accessed from College Referral Link**
   - URL includes `?ref=college_123` or similar parameter
   - Signup button and form inherit this parameter
   - Source attribution passed to signup endpoint

6. **Homepage Accessed from Affiliate Link**
   - URL includes `?utm_source=affiliate_partner&utm_medium=referral`
   - Parameters preserved and passed to signup
   - Analytics attribute signup to correct affiliate

7. **Institutional Partnership Information Not Yet Available**
   - "For Institutions" section either:
     - Not displayed (if partnerships not yet available)
     - Or shows "Coming soon" message
     - Or links to contact form for institutional inquiries
   - Product team can toggle visibility via configuration

8. **Homepage Hero Image Fails to Load**
   - Solid background color shows instead
   - Text remains readable and page functional
   - Alt text on image displayed if applicable

9. **User Bookmarks Homepage**
   - Page remains same URL (no redirects)
   - Bookmarked link always works
   - No session-dependent content

10. **User Shares Homepage on Social Media**
    - Open Graph meta tags defined:
      - og:title, og:description, og:image, og:url
    - Twitter card meta tags (if applicable)
    - Social share preview shows platform name, headline, and relevant image

## Error States

### Error 1: Page Fails to Load

**Trigger:** Server error or network timeout

**Display:**
- 500 error page with message: "Something went wrong. Please try again later."
- "Back to home" link or refresh button

---

### Error 2: Images Fail to Load

**Trigger:** CDN or image hosting unavailable

**Display:**
- Solid background color or placeholder instead of image
- Text remains readable
- Alt text provides context if hover over missing image

---

### Error 3: CMS Config Fails to Fetch (if using CMS)

**Trigger:** API /api/homepage-config returns error

**Display:**
- Page loads with hardcoded fallback copy
- Site still functional, no blank sections
- Error logged server-side for monitoring
