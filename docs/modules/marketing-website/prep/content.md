# Content Definition: Marketing Website

This document defines all copy required for the NicheLearner marketing website. Copy is organized by page/section, form, notification type, and error state.

**Tone:** Credible, direct, student-focused. Emphasize honest feedback, personalization, and clear next steps. Avoid jargon; use language 10th–12th grade students and parents understand. Build trust through transparency.

## Error Messages

All errors use plain language and explain what went wrong and what to do next.

### Email Validation

**Error:** (User enters invalid email format)

```
Please enter a valid email address (e.g., name@example.com)
```

---

### Required Field Missing

**Error:** (User submits form with blank required field)

```
[Field Name] is required. Please fill it in.
```

**Example:**
```
Email is required. Please fill it in.
```

---

### Password Too Weak

**Error:** (Password does not meet minimum requirements)

```
Your password needs at least 8 characters and should include a mix of letters, numbers, and symbols.
```

---

### Passwords Don't Match

**Error:** (Password and confirmation password fields differ)

```
Your passwords don't match. Please check and try again.
```

---

### Email Already in Use

**Error:** (User tries to sign up with an email already registered)

```
This email address is already linked to a NicheLearner account. 
[Recover your account →] or [Use a different email →]
```

---

### Network/Submission Error

**Error:** (Form submission fails due to server or network issue)

```
Oops, something went wrong. Please try again. 
If the problem persists, contact us at support@nichelearner.com
```

---

### Payment Failed (Trial-to-Paid)

**Error:** (Payment processor declines transaction)

```
Your payment couldn't be processed. Please check your card details and try again, or choose a different payment method.
```

---

### Institutional Form: Missing Required Field

**Error:** (College inquiry form submitted without required fields)

```
Please fill in all required fields: College Name, Number of Students, Your Name, Your Email, and Phone Number.
```

## Tone Guidelines

When writing any copy not explicitly defined above:

1. **Be direct.** Avoid marketing fluff. Say what the product does and why it matters.
2. **Use student/parent language.** Avoid jargon. If you must use technical terms, explain them.
3. **Emphasize honest feedback, not perfection.** The value is clarity, not shortcuts.
4. **Build trust through transparency.** Explain security, data handling, and what costs money (vs. free).
5. **Make CTAs clear.** Buttons should say what happens next, not "Click Here."
6. **Anticipate friction.** Help users overcome common hesitations (no credit card needed, can cancel anytime).
